# Principios de diseño UX/UI

El diseño del prototipo del TFM no se considera definitivo. Se definen estos principios:

1. Claridad: el usuario debe entender fácilmente dónde está la FAQ, dónde está el chat y qué hace cada cosa.
2. Velocidad: debe ser más rápido consultar OperaBot que preguntar a una persona.
3. Confianza:
   - Mostrar fuentes de la información.
   - Indicar cuándo la IA no está segura.
   - Permitir escalar a una persona si es necesario.
4. Baja fricción:
   - Navegación simple, sin menús escondidos.
   - Buen contraste, tipografía legible.
   - Evitar sobrecargar de opciones en la pantalla principal.

Estos principios guiarán el diseño futuro de pantallas y flujos, por encima del prototipo actual del TFM.
